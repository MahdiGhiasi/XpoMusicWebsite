Enabling proxy loopback for Xpotify

Windows 10 UWP apps are by default not allowed to communicate through a local proxy for security reasons. If you want to route Xpotify traffic (or any other UWP app) to a local proxy server, you'll need to enable proxy loopback for that specific app.

To do so, you'll need to download and install Fiddler.

After installing Fiddler, open it. You'll see an option called WinConfig on the top left corner. (See the picture below)

After clicking on the WinConfig button, AppContainer Loopback Exemption Utility opens.

Find Xpotify from the list, select it and make sure the checkbox on the left side is checked. Then click on Save Changes button.

Finally, close Fiddler and Xpotify, and then reopen Xpotify.
